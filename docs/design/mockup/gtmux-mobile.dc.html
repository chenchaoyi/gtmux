推送与连接 — 离开工位也能收到

杀手级时刻：你去倒咖啡，agent 卡在权限上——锁屏震一下。推送走 gtmux serve → relay → APNs，由 Apple 投递，手机离开 VPN 也能收到；点开后需要内网才拉实况。

delivery path
gtmux serve → push relay → APNs → 📱 设备
Mac 把 alert（waiting / done）转给 relay，relay 持 APNs key、保持无状态。设备 token 存在 Mac（~/.config/gtmux/push-tokens.json），通过 POST /api/push/register 注册。iOS 用原生 APNs token，不需要 Firebase。
behavior
waiting任意态 → waiting（需要你）。锁屏推送 + 前台时改为应用内红色 banner。
doneworking → idle（跑完了）。同样可推送，前台绿色 banner。
tap点推送 → 深链直达该 pane 的 Detail（读 payload 里的 pane），等于 gtmux focus --last 的「跳过去」。

why split-view
手机是「列表 → 进详情」两步；iPad 一屏并置：左栏 320pt 常驻雷达，右区实时终端 + composer。少切换、全局可见，贴合 Mail / Messages 的 iPad 范式。
adaptive · size class
横屏 / 宽（regular）：双栏并置（如上）。
竖屏 / 分屏变窄：侧栏退为可滑出抽屉，详情占满；进一步压缩（Slide Over）回退到手机式单列堆叠。
同一套 RN 组件按 useWindowDimensions 断点切换，不另起一套。

· 硬件键盘：↑↓ 选 agent、⏎ focus、⌘1–9 直达、⌘F 搜索。
· Stage Manager / 多任务下窗口可变宽，按断点自适应。
· 字号 A−/A+ 与回滚缓冲沿用；终端在大屏可显更多列。
