gtmux · web 浏览器镜像 · 工作台版

大屏工作台,不是放大的手机

浏览器屏大、有真键鼠——所以不照搬手机单列。左侧是 session / window / pane 目录,把任意 pane 拖到画板自由排列、缩放,像 tmux 那样自己组合视图。仍是只读镜像:你编排的是「自己的观察布局」,不改动真实 tmux 树。底层全部复用现有契约(/api/agents · /api/pane · /api/transcript · /api/diff · SSE),无需新后端。

04

设计判断 — 合理性与边界

✓ 为什么合理
gtmux 的核心价值就是同时盯多个 agent。大屏单列要反复进出才能对比;多 pane 画板让「等你的 / 在跑的 / 刚完成的」一屏并置,正合「雷达」定位。
✓ 零新后端
多 pane = 多个并发 /api/pane?id 挂载;目录树来自 /api/agents 的 session/window/pane;SSE 驱动状态变化。契约已全有。
⚠ 唯一边界:只读
真 tmux 能 split/kill/spawn,镜像不能假装。「编排窗口与 pane」= 你自己的观察布局,不改动真实 tmux 树。要输入回手机/Mac。
05

能力清单(含建议新增)

① session/window/pane 目录
左侧树按状态分组(needs-you→working→idle),window 可展开到 pane;搜索过滤;拖到画板。侧栏可一键收起(⇤,留一个带 waiting 计数的小标签可重新展开)、可拖右缘自由调宽(记住宽度);窄屏自动收起。这就是你说的「agent session 目录」。
② 平铺布局 · 拖隔断自适应
像 tmux 一样平铺:拖动两块 tile 之间的分隔线,相邻 pane 同步自适应(不重叠、不留白)。拖进新 pane 自动分割出位置;移走后其余填满。也可切「自由浮动」模式(叠放/任意位置)。每块是独立的实时 xterm 镜像。
③ 保存布局 / 预设新增
「Frontend trio」「全部 waiting」等命名布局,存 localStorage;重开链接即恢复。顶栏一键切换。
④ 自动浮出 waiting新增
可选:某 pane 翻成 waiting 时自动浮上画板并脉冲,画板即雷达——不必盯着目录。SSE alert 驱动。
⑤ 点击聚焦 · 最大化
点某块 → 放大到满画板(其余收为底部缩略条/淡出);再点或 Esc 还原平铺。双击 / ⤢ 则进全屏单 pane(细读、滚历史、看 diff)。等于把现有单列视图作为 focus。
⑥ 每 tile · 终端/对话/diff新增
tile 头部切 终端(xterm 镜像)/ 对话(解析对话)/ diff(/api/diff 改动)。与手机双模式同名(终端↔对话)。

08

输入能力 + 权限透出

网页从只读镜像升级为可输入(POST /api/send / attach)。但输入受 owner/guest scope 严格约束——每个 tile 头部要明示本 pane 是「可敲」还是「只读」,guest 尤其不能有歧义。

每 tile 明示权限
· ⌨ 可输入(青):有 composer + 1/2/3 快捷键。
· 👁 只读(灰):无输入区,底部一行「未授权」说明,不留空文本框骗点击。
· owner 全 pane 可输入;guest 仅该分享链接勾选的「输入」范围(逐链接 scope,输入⊆可见),且要 host「允许协作者输入」总开关。
服务端强制
网页只镜像权限,真正的门在服务端:非白名单 pane 的 /api/send 直接拒。guest 看不到、也拖不出不在其链接「可见」范围里的 pane。撤销/过期即刻生效。
owner vs guest 一眼分
owner(pair 链接)顶栏显「全权 · home-mac」;guest(share 链接)顶栏显「协作视图 · 来自 Ada」+ 只出现被授权的 pane,连 HQ 指挥台也不对 guest 开放。
