05

空状态 & 首次运行

没有 agent 时不报错、不留白尴尬，给一条可复制的启动命令（任意 coding agent，不限 Claude）。首次运行的解释卡先讲核心价值——一键跳到对应的 tmux pane，再说明为此需要一次性的「自动化」权限（通过 AppleScript 把对应终端标签页带到最前），把「为什么、给谁、怎么撤销」说清楚，而不是把系统弹窗直接糊用户脸上。

首次运行 · 自动化权限

跳转需要「自动化」权限

点击某个 agent 时，gtmux 用 AppleScript 把它所在的终端标签页和 tmux pane 切到最前。这需要一次「自动化」授权，只切换窗口，不读取终端内容。

1
点下方 允许并继续，会弹出 macOS 系统对话框
2
在「“gtmux” 想要控制 “Ghostty”」中点 好
3
随时可在 系统设置 › 隐私与安全性 › 自动化 撤销（原生终端下的 agent 同理，授权对应终端即可）
不授权也能用：agents、overview 照常工作，只是不能点击跳转。
