移动端有完整的通知 / 灵动岛 / Live Activity 设计,但菜单栏一直没有 spec macOS 系统通知。当你在别的全屏 app 里工作、没看菜单栏时,通知是第二张安全网——且 UNNotificationAction 支持按钮 + 行内文字回复,正好接同一套 1/2/3。

两类通知
waiting带 1/2/3 动作按钮 + 行内文字回复;点按钮即 send,无需打开 app。

规则
· 走 UNUserNotificationCenter,UNNotificationAction 承载 1/2/3,UNTextInputNotificationAction 承载自定义回复。
· 与「专注模式」协作:waiting 设为时效性(time-sensitive)可穿透;done 普通级。
· 偏好里「通知」开关已存在,新增动作按钮与勿扰时段(对齐移动端设置)。
· 通知去重:同一 pane 状态没变不重复打扰;解决后自动撤回。
三屏一致
菜单栏弹层(§09)· macOS 通知(§10)· 手机通知/Live Activity —— 全部用同一套 1/2/3 + 真实 label + 分行。一处解析、三处复用,这是 gtmux 的核心差异点:在哪都能一键解阻塞。
